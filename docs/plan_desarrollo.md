# Plan de desarrollo

Para desarrollar el sistema AnimeFull se trabajara por partes, para poder llevar un avance mas ordenado en GitHub.

Primero se realizo la creacion del repositorio, el README y la estructura de carpetas del proyecto.

Despues se empezo con la documentacion del sistema, donde se explicaron las herramientas, la arquitectura, los agentes, las reglas de inferencia, la base de datos, la interfaz visual, el flujo y los requisitos.

Despues de terminar la carpeta docs, se comenzara con la parte de codigo.

Lo siguiente sera crear los archivos principales como main.py y requirements.txt.

Luego se empezara a trabajar la interfaz visual con Streamlit, para que el sistema tenga una pantalla principal parecida a un sistema de tienda.

Despues se hara la conexion con la base de datos usando SQLite.

Luego se agregaran los modulos de productos, inventario, pedidos, clientes y alertas.

Tambien se programaran las reglas de inferencia para detectar bajo stock, productos agotados y recomendaciones.

Mas adelante se agregaran los agentes del sistema, como el agente cliente, inventario, recomendador, pedidos, reabastecimiento y explicador.

Al final se haran pruebas, mejoras visuales y correcciones para que el proyecto quede mas completo y presentable visualmente y informaticamente.

La idea es hacer commits pequeños durante los dias de desarrollo, para mostrar el avance del proyecto paso por paso y tener un mejor orden, en caso de que se tenga que estar realizando cambios, en los codigos, los commits lo mostraran.
